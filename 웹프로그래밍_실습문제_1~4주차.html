h1~h6 태그의 특징

다음 중 HTML 제목 태그(h1~h6)에 대한 설명으로 올바른 것은?

① h6이 h1보다 더 크게 표시된다.
② h1은 인라인(inline) 요소이다.
③ 한 페이지에 h1은 하나만 사용하는 것이 권장된다.
④ h 태그는 SEO와 관계없다.
정답 ③ — h1이 가장 크고 h6이 가장 작습니다. h1은 블록 요소이며 SEO에 영향을 줍니다.
객관식하 2번

텍스트 서식 태그 — strong vs b

<strong>과 <b>의 차이로 올바른 것은?

① 두 태그는 기능과 의미가 완전히 동일하다.
② strong은 의미론적 강조로 스크린리더가 읽고, b는 시각적 효과만이다.
③ b 태그는 HTML5에서 삭제되었다.
④ strong은 인라인 요소가 아니다.
정답 ② — strong/em은 의미론적 강조(접근성), b/i는 시각적 효과만 담당합니다.

목록 태그 — ul / ol / dl

다음 중 <dl> 태그에 대한 설명으로 올바른 것은?

① dl은 순서 있는 목록을 만들 때 사용한다.
② dl 안에 <li> 태그를 사용해야 한다.
③ dl은 용어-정의 쌍(dt=용어, dd=설명)으로 이루어진 목록이다.
④ dl 태그는 이미지 목록에만 사용한다.
정답 ③ — dl은 정의형 목록으로 dt(용어)와 dd(설명)를 함께 사용합니다.

객관식중 5번

table — thead / tbody / tfoot

HTML 표에서 <tfoot>에 대한 설명으로 올바른 것은?

① tfoot는 표의 본문 데이터를 담는 태그이다.
② tfoot는 thead보다 먼저 코드에 작성해야 화면에 하단에 표시된다.
③ tfoot는 표의 푸터 행을 의미론적으로 구분하며 합계/주석 등에 사용된다.
④ tfoot 안에는 td 태그를 사용할 수 없다.
정답 ③ — tfoot는 표 하단을 의미론적으로 구분하며, td 사용이 가능합니다. 코드 순서와 화면 표시 위치는 브라우저가 자동 처리합니다.

② <nav> — 사이트 내비게이션 링크 모음

display — block vs inline

display: inline에 대한 설명으로 올바른 것은?

① 한 줄 전체를 차지한다.
② width, height 지정이 무시된다.
③ margin-top/bottom이 적용된다.
④ div 태그의 기본값이다.
정답 ② — inline은 내용 크기만큼만 차지하며 width/height/상하margin이 무시됩니다.
객관식하2번

position — static

position의 기본값은 무엇이며, 이 값일 때 top 속성은 어떻게 되는가?

① relative / top이 적용된다.
② static / top이 무시된다.
③ absolute / top이 뷰포트 기준으로 적용된다.
④ fixed / top이 부모 기준으로 적용된다.
정답 ② — static이 기본값이며, top/left 등의 위치 속성이 무시됩니다.

position: absolute 요소의 위치 기준점은?

② 항상 body 태그 기준

④ 자기 자신의 원래 위치 기준
정답 ③ — absolute는 가장 가까운 non-static 조상을 기준으로 배치됩니다.
객관식중5번

float 해제 (clearfix)

float 요소로 인한 부모의 높이 붕괴를 해결하는 방법으로 올바르지 않은 것은?

① 부모에 overflow: hidden 적용
② 부모::after { content:""; display:table; clear:both; } clearfix 패턴
③ 마지막 자식에 <div style="clear:both"></div> 추가
④ 부모에 float: none 적용
정답 ④ — float:none은 해당 요소의 float을 제거하지만 부모 높이 붕괴를 해결하지 않습니다.
객관식중6번

z-index 동작 조건

z-index 속성이 동작하려면 반드시 필요한 조건은?

① display: block이어야 한다.
② position이 static이 아닌 값(relative/absolute/fixed/sticky)이어야 한다.
③ 음수 z-index는 사용할 수 없다.
④ 요소가 float 상태여야 한다.
정답 ② — position이 static이면 z-index가 무시됩니다. 음수 z-index도 사용 가능합니다.
객관식상7번

visibility vs display:none

visibility: hidden과 display: none의 차이로 올바른 것은?

① visibility:hidden은 공간을 유지하며 숨기고, display:none은 공간도 없앤다.
② 두 속성 모두 공간을 유지하며 요소를 숨긴다.
③ display:none은 공간을 유지하고, visibility:hidden은 공간을 없앤다.
④ 두 속성 모두 DOM에서 요소를 제거한다.
정답 ① — visibility:hidden은 공간 유지, display:none은 공간까지 제거합니다.

객관식하2번

transition vs animation

transition과 @keyframes animation의 차이로 올바른 것은?

① transition은 무한 반복이 가능하고 animation은 불가능하다.
② 둘 다 hover 없이 자동 실행된다.
③ transition은 hover 등 상태 변화가 필요하고, animation은 자동 실행 가능하다.
④ transition은 중간 단계(50% 등)를 여러 개 지정할 수 있다.
정답 ③ — transition은 상태 트리거 필요, animation은 자동 실행·무한 반복·다단계 설정이 가능합니다.

h1~h6 태그의 특징

다음 중 HTML 제목 태그(h1~h6)에 대한 설명으로 올바른 것은?

① h6이 h1보다 더 크게 표시된다.
② h1은 인라인(inline) 요소이다.
③ 한 페이지에 h1은 하나만 사용하는 것이 권장된다. ✅
④ h 태그는 SEO와 관계없다.
해설: h1이 가장 크고 h6이 가장 작습니다. h1은 블록(block) 요소이며, 검색엔진 최적화(SEO)에 직접적인 영향을 줍니다. 한 페이지에 h1은 하나만 사용하는 것이 표준 권장사항입니다.

텍스트 서식 태그 — strong vs b

<strong>과 <b>의 차이로 올바른 것은?

① 두 태그는 기능과 의미가 완전히 동일하다.
② strong은 의미론적 강조로 스크린리더가 읽고, b는 시각적 효과만이다. ✅
③ b 태그는 HTML5에서 삭제되었다.
④ strong은 인라인 요소가 아니다.
해설: <strong>/<em>은 의미론적(semantic) 강조 태그로 스크린리더가 강조해서 읽습니다. <b>/<i>는 시각적 효과(굵게/기울임)만 있는 표현용 태그입니다. 둘 다 인라인 요소입니다.

목록 태그 — ul / ol / dl

다음 중 <dl> 태그에 대한 설명으로 올바른 것은?

① dl은 순서 있는 목록을 만들 때 사용한다.
② dl 안에 <li> 태그를 사용해야 한다.
③ dl은 용어-정의 쌍(dt=용어, dd=설명)으로 이루어진 목록이다. ✅
④ dl 태그는 이미지 목록에만 사용한다.
해설: <dl>(Definition List)은 <dt>(Definition Term, 용어)와 <dd>(Definition Description, 설명)를 쌍으로 사용합니다. <li>는 ul/ol에만 사용합니다.

table — thead / tbody / tfoot

HTML 표에서 <tfoot>에 대한 설명으로 올바른 것은?

① tfoot는 표의 본문 데이터를 담는 태그이다.
② tfoot는 thead보다 먼저 코드에 작성해야 화면에 하단에 표시된다.
③ tfoot는 표의 푸터 행을 의미론적으로 구분하며 합계/주석 등에 사용된다. ✅
④ tfoot 안에는 td 태그를 사용할 수 없다.
해설: <tfoot>은 표의 요약/합계 행을 의미론적으로 묶는 태그입니다. <td> 사용이 가능합니다. 코드 작성 순서와 무관하게 브라우저가 항상 하단에 렌더링합니다. 본문 데이터는 <tbody>에 담습니다.

② 3개 (colspan 값)

display — block vs inline

display: inline에 대한 설명으로 올바른 것은?

① 한 줄 전체를 차지한다. (→ block의 특성)
② width, height 지정이 무시된다. ✅
③ margin-top/bottom이 적용된다. (→ inline은 상하 margin 무시)
④ div 태그의 기본값이다. (→ div 기본값은 block)
해설: inline은 내용 크기만큼만 차지하며 width·height·상하 margin이 무시됩니다. 한 줄 전체를 차지하는 것은 block입니다. span, a, em 등이 기본 inline 요소입니다.
객관식하2번

position — static

position의 기본값은 무엇이며, 이 값일 때 top 속성은 어떻게 되는가?

① relative / top이 적용된다.
② static / top이 무시된다. ✅
③ absolute / top이 뷰포트 기준으로 적용된다.
④ fixed / top이 부모 기준으로 적용된다.
해설: position 기본값은 static이며, 이 상태에서는 top/left/right/bottom 및 z-index 속성이 모두 무시됩니다.

border-radius

정사각형 div를 완전한 원형으로 만들려면 border-radius 값은?

① border-radius: 10px; (약간 둥글어짐)
② border-radius: 100px; (크게 둥글어지지만 박스 크기에 따라 다름)
③ border-radius: 50%; ✅
④ border-radius: round; (존재하지 않는 값)
해설: 정사각형에 border-radius: 50%를 적용하면 완전한 원이 됩니다. %는 요소의 너비/높이에 상대적이므로 크기에 관계없이 항상 원이 됩니다.
객관식중4번

position absolute의 기준점

position: absolute 요소의 위치 기준점은?

① 항상 브라우저 화면(뷰포트) 기준 (→ fixed의 특성)
② 항상 body 태그 기준
③ 가장 가까운 position이 static이 아닌 조상 요소 기준 ✅
④ 자기 자신의 원래 위치 기준 (→ relative의 특성)
해설: absolute는 부모 중 position: relative/absolute/fixed/sticky가 지정된 가장 가까운 조상을 기준으로 배치됩니다. 해당 조상이 없으면 body(초기 블록)가 기준이 됩니다.
객관식중5번

float 해제 (clearfix)

float 요소로 인한 부모의 높이 붕괴를 해결하는 방법으로 올바르지 않은 것은?

① 부모에 overflow: hidden 적용
② 부모::after { content:""; display:table; clear:both; } clearfix 패턴
③ 마지막 자식에 <div style="clear:both"></div> 추가
④ 부모에 float: none 적용 ✅ (잘못된 방법)
해설: float: none은 해당 요소의 float을 해제하지만 부모의 높이 붕괴를 해결하지 못합니다. 올바른 해결책은 ①②③ 중 하나입니다. 가장 권장되는 방법은 clearfix 패턴(::after)입니다.
객관식중6번

z-index 동작 조건

z-index 속성이 동작하려면 반드시 필요한 조건은?

① display: block이어야 한다.
② position이 static이 아닌 값(relative/absolute/fixed/sticky)이어야 한다. ✅
③ 음수 z-index는 사용할 수 없다. (→ 음수도 사용 가능)
④ 요소가 float 상태여야 한다.
해설: z-index는 position: static(기본값)인 요소에는 적용되지 않습니다. 음수 z-index도 사용 가능하며, 값이 낮을수록 배경쪽으로 이동합니다.
객관식상7번

visibility vs display:none

visibility: hidden과 display: none의 차이로 올바른 것은?

① visibility:hidden은 공간을 유지하며 숨기고, display:none은 공간도 없앤다. ✅
② 두 속성 모두 공간을 유지하며 요소를 숨긴다.
③ display:none은 공간을 유지하고, visibility:hidden은 공간을 없앤다. (반대)
④ 두 속성 모두 DOM에서 요소를 제거한다.
해설: visibility: hidden은 보이지 않지만 공간은 그대로 유지되어 레이아웃에 영향을 줍니다. display: none은 요소가 완전히 사라져 공간도 없어집니다. 두 속성 모두 DOM에서 요소를 제거하지는 않습니다.
✏️ 단답식 (8~10번)
단답식하8번

정답inline-block

단답식중9번

sticky 위치 기준

position: sticky에서 헤더를 화면 최상단에 붙이려면 어떤 속성을 0으로 설정해야 하는가?

정답top
해설: position: sticky; top: 0;으로 설정하면 스크롤 전에는 원래 위치에 있다가, 스크롤로 화면 상단(top: 0)에 닿는 순간 고정됩니다. 탭 내비게이션이나 헤더에 자주 사용되는 패턴입니다.
단답식상10번

float 높이 붕괴 해결 기법

CSS animation을 무한 반복시키려면 animation-iteration-count에 어떤 값을 쓰는가?

정답infinite
해설: animation-iteration-count: infinite로 무한 반복합니다. 숫자(예: 3)를 쓰면 해당 횟수만큼만 반복합니다. 단축 속성에서는 animation: 이름 시간 infinite으로 씁니다.

keyframe 끝 키워드

@keyframes에서 0%는 from으로 대체할 수 있다. 100%에 해당하는 키워드는?

정답to
해설: @keyframes { from { ... } to { ... } }는 @keyframes { 0% { ... } 100% { ... } }와 동일합니다. 중간 단계가 없는 단순 애니메이션에서 from/to가 더 읽기 쉽습니다.

요소 기울이기 함수

transform에서 요소를 X/Y축으로 기울이는 함수 이름은?

정답skew
해설: skew(xDeg, yDeg)로 X/Y 방향으로 기울입니다. skewX(deg)는 X축만, skewY(deg)는 Y축만 기울입니다. 평행사변형 모양이나 이탤릭 효과에 활용됩니다.

코드블럭하11번

빈칸 — transition + hover scale

정답시간 예: 0.3s  /  hover  /  1.3
.btn { transition: all 0.3s ease; } .btn:hover { transform: scale(1.3); }
해설: 시간 값(0.2s ~ 1s 등)은 어느 것이든 정답으로 처리됩니다. hover와 scale(1.3)이 핵심입니다. transition이 없으면 즉시(뚝) 변환되고, 있으면 지정된 시간 동안 부드럽게 변환됩니다.

빈칸 — @keyframes 기본

정답keyframes  /  infinite
@keyframes moveRight { 0% { margin-left: 0; } 100% { margin-left: 300px; } } .box { animation: moveRight 2s infinite alternate; }
코드블럭중13번

빈칸 — 복합 transform

정답translate  /  rotate
.box { transition: all 0.4s ease; } .box:hover { transform: translate(20px, 0) rotate(45deg); }
해설: transform에 여러 함수를 공백으로 구분해 나열하면 복합 변환이 됩니다. 적용 순서는 오른쪽에서 왼쪽으로 처리되므로 순서에 따라 결과가 달라질 수 있습니다.
코드블럭상14번

빈칸 — 공이 튀는 애니메이션

정답ease-in  /  alternate
@keyframes fall { 0% { margin-top: 0; height: 60px; } 100% { margin-top: 60px; height: 35px; width: 75px; } } .ball { animation: fall 1s ease-in infinite alternate; }
해설: ease-in은 천천히 시작해 빠르게 끝나는 속도 곡선으로 중력 가속을 표현합니다. alternate로 튀어오르는 역방향 동작도 자연스럽게 처리됩니다.

hover 시 색상+크기 변화 (transition)

모범 답안
<style> .box { width: 100px; height: 100px; background: #ef4444; border-radius: 8px; transition: all 0.3s ease; } .box:hover { background: #3b82f6; transform: scale(1.2); } </style> <div class="box"></div>

좌우로 움직이는 @keyframes 박스

모범 답안

채점 기준: @keyframes + animation: 이름 시간 + infinite + alternate 가 있으면 정답입니다.

4종 transform 카드 (hover 적용)

모범 답안
<style> .wrap { display: flex; gap: 10px; } .box { width: 70px; height: 70px; color: white; font-weight: 700; display: flex; align-items: center; justify-content: center; border-radius: 8px; transition: all 0.35s ease; cursor: pointer; } .t1 { background: #ec4899; } .t2 { background: #f59e0b; } .t3 { background: #3b82f6; } .t4 { background: #22c55e; } .t1:hover { transform: translate(20px, 20px); } .t2:hover { transform: scale(1.5); } .t3:hover { transform: rotate(45deg); } .t4:hover { transform: skew(20deg); } </style> <div class="wrap"> <div class="box t1">이동</div> <div class="box t2">크기</div> <div class="box t3">회전</div> <div class="box t4">기울임</div> </div>
채점 기준: transition + :hover + translate, scale, rotate, skew 4가지가 모두 있으면 정답입니다.

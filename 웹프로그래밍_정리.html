p 태그 — 단락(Paragraph)

p는 하나의 문단을 나타내는 블록 요소. 위아래에 자동으로 여백(margin)이 생기며, 연속으로 써도 각각 독립된 단락으로 분리됨. 줄바꿈이 필요할 때 br을 쓰는 것과 달리, p는 의미론적 단락 구분에 사용.

display · position · float · z-index · border · visibility

주간 요약 CSS의 핵심인 박스 모델과 레이아웃 시스템을 이해합니다. display 속성(block/inline/inline-block), position 5종류, float/clear, z-index 스태킹을 실습합니다.

display: block vs inline vs inline-block

block은 한 줄 전체를 차지하고 width/height/margin 모두 적용 가능(div, p 기본값). inline은 내용 크기만큼만 차지하며 width·height·상하margin이 무시됨(span, a 기본값). inline-block은 한 줄 배치이면서 width/height/margin 모두 적용 가능.

코드 & 결과
/* block — 한 줄 차지 */ div { display: block; } /* inline — 내용 크기만큼만 width/height 지정 불가 */ span { display: inline; } /* inline-block — 한 줄 + 크기 지정 */ div { display: inline-block; width: 80px; height: 50px; }

position: relative / absolute

relative는 원래 위치 기준으로 이동하며 원래 공간은 그대로 유지됨. absolute는 가장 가까운 position이 지정된 부모를 기준으로 배치되며 문서 흐름에서 빠져나옴. static(기본값)은 top/left가 무시됨.

코드 & 결과
/* 부모: relative */ .parent { position: relative; } /* 자식: relative — 원위치 기준 이동 공간은 그대로 남음 */ .child-relative { position: relative; top: 40px; left: 60px; } /* 자식: absolute — 부모 기준 문서 흐름에서 분리됨 */ .child-absolute { position: absolute; bottom: 8px; right: 8px; }
결과 미리보기 (점선 = 부모 position:relative)

position: fixed & sticky

fixed는 뷰포트(화면) 기준으로 고정되어 스크롤해도 같은 위치에 머뭄. sticky는 스크롤 전엔 relative처럼 동작하다가 지정한 임계점(top: 0 등) 도달 시 fixed처럼 고정됨.

코드 & 결과
/* fixed — 뷰포트 기준 고정 스크롤해도 화면에 그대로 */ #fixedBtn { position: fixed; bottom: 20px; right: 20px; } /* sticky — 스크롤 중 임계점 도달 시 고정 relative + fixed 혼합 */ .sticky-header { position: sticky; top: 0; background: yellow; }

z-index 스태킹

겹치는 요소의 앞뒤 순서를 숫자로 지정. 숫자가 클수록 앞에 표시됨. ⚠️ position이 static이 아닌 값이어야 동작함(relative/absolute/fixed/sticky). 부모-자식 관계에서 쌓임 맥락(stacking context)에 주의.

코드 & 결과
/* ⚠️ position이 있어야 z-index 동작! */ div { position: absolute; } .red { z-index: 1; } /* 가장 뒤 */ .green{ z-index: 20; } /* 중간 */ .blue { z-index: 30; } /* 맨 앞 */

float & clear

float: left/right로 요소를 띄워 텍스트가 그 주변을 감싸게 함. clear: left=왼쪽 float 해제, clear: right=오른쪽 float 해제, clear: both=양쪽 모두 해제. float 요소가 많으면 부모의 높이가 0이 되는 "높이 붕괴" 문제 → 부모에 ::after { content:""; display:table; clear:both; } clearfix 패턴으로 해결.

코드 & 결과
.box { float: left; margin-right: 10px; } /* clearfix — 부모 높이 붕괴 방지 */ .parent::after { content: ""; display: table; clear: both; }

border 스타일

visibility vs display:none

visibility: hidden은 요소를 보이지 않게 하지만 원래 차지하던 공간은 그대로 유지. display: none은 요소를 완전히 제거해 공간도 사라짐. 레이아웃을 유지하면서 숨길 때는 visibility, 공간까지 없앨 때는 display: none 사용.

코드 & 결과
/* 숨기기 — 공간 유지 */ .hidden { visibility: hidden; } /* 숨기기 — 공간도 사라짐 */ .gone { display: none; }

3주차 — 목록 스타일 & 표 디자인

list-style · 네비게이션 메뉴 · 표 CSS · 시멘틱 HTML5
03
주간 요약 CSS로 목록(list)을 수평 내비게이션 메뉴로 변환하는 방법, 표(table) 스타일링(줄무늬, 호버, 헤더/푸터 구분), 시멘틱 HTML5 레이아웃(nav, section, article, footer)을 학습합니다.

list-style-type 마커 종류

list-style-type으로 불릿 마커 모양을 지정. ul은 disc·circle·square·none, ol은 decimal·upper-roman·lower-alpha 등. list-style: none + padding 제거로 기본 마커를 없애고 커스텀 디자인의 메뉴를 만들 수 있음.

코드 & 결과
ul.disc { list-style-type: disc; } ul.circle { list-style-type: circle; } ul.square { list-style-type: square; } ul.none { list-style-type: none; } ol.roman { list-style-type: upper-roman; } ol.alpha { list-style-type: lower-alpha; } ol.decimal{ list-style-type: decimal; }

수평 내비게이션 메뉴

ul을 수평 메뉴로 만드는 핵심 3가지: ① list-style: none으로 불릿 제거 ② li { display: inline-block }으로 한 줄 배치 ③ a { text-decoration: none }으로 밑줄 제거. a:hover로 색상 변경 효과 추가.

코드 & 결과
<nav id="menubar"> <ul> <li><a href="#">Home</a></li> <li><a href="#">Espresso</a></li> <li><a href="#">Latte</a></li> </ul> </nav> #menubar ul { list-style: none; } #menubar li { display: inline-block; } #menubar a { text-decoration: none; color: white; padding: 0.4rem 0.8rem; } #menubar a:hover { color: violet; }

줄무늬 표 + hover 효과

thead(헤더)/tbody(본문)/tfoot(푸터)으로 표 구조를 의미론적으로 분리. tbody tr:nth-child(even)으로 짝수 행에 배경색을 주면 줄무늬 효과, tbody tr:hover로 마우스 올린 행 강조.

코드 & 결과 (표에 마우스를 올려보세요)
table { border-collapse: collapse; } thead { background: #333; color: yellow; } tfoot { background: #f1f5f9; } /* 짝수 행 배경 */ tbody tr:nth-child(even) { background: lightblue; } /* 행 hover */ tbody tr:hover { background: pink; }

4주차 — CSS Transform & Animation

2D 변환 · transition · @keyframes · 다양한 애니메이션
04
주간 요약 CSS의 2D transform 함수(translate, scale, rotate, skew)와 transition으로 부드러운 전환 효과, 그리고 @keyframes를 이용한 자동 반복 애니메이션을 학습합니다.

transform 즉시 변환 — transition 없음 (transform.html)

transform은 요소의 모양/위치를 변환하는 CSS 속성. transition이 없으면 hover 시 즉시(뚝) 변환됨. translate(x,y)=이동, scale(n)=크기, rotate(deg)=회전, skew(x,y)=기울임. 실제 레이아웃에는 영향을 주지 않음.

코드 & 결과 — 마우스를 올리면 즉시 변환됩니다
/* transition 없음 → 즉시 변환 */ .translate { background-color: pink; } .translate:hover { transform: translate(80px, 80px); } .scale { background-color: lightgoldenrodyellow; } .scale:hover { transform: scale(2, 2); } .rotate { background-color: lightblue; } .rotate:hover { transform: rotate(45deg); } .skew { background-color: lightgreen; } .skew:hover { transform: skew(30deg, 30deg); }

transform 4종류 (박스에 마우스를 올려보세요)

transition: all 시간 timing을 추가하면 transform 변환이 부드럽게 이어짐. ease는 처음·끝이 느리고 중간이 빠름, linear는 일정 속도, ease-in은 천천히 시작, ease-out은 천천히 끝남.

코드 & 결과 — transition: all 0.35s 적용됨
/* transition으로 부드럽게 */ .box { transition: all 0.35s ease; } /* 1. 이동 */ .translate:hover { transform: translate(20px, 20px); } /* 2. 크기 */ .scale:hover { transform: scale(1.5); } /* 3. 회전 */ .rotate:hover { transform: rotate(45deg); } /* 4. 기울임 */ .skew:hover { transform: skew(20deg, 5deg); }

@keyframes 자동 애니메이션 (자동 재생 중)

@keyframes 이름 { 0%{...} 100%{...} }으로 애니메이션 정의 후 animation: 이름 시간 반복 방향으로 적용. infinite=무한 반복, alternate=정방향→역방향 반복. transition과 달리 hover 없이 자동 실행되고 중간 단계(50% 등)를 여러 개 지정 가능.

코드 & 결과
/* 1. 좌우로 이동하는 박스 */ @keyframes moveRight { 0% { margin-left: 0; } 100% { margin-left: 300px; } } .box { animation: moveRight 2s infinite alternate; } /* 2. 원으로 변하며 날아가는 박스 */ @keyframes fly { 0% { border-radius: 0; } 100% { border-radius: 50%; background: coral; } } /* 3. 공이 떨어지는 효과 */ @keyframes fall { 0% { margin-top: 0; height: 60px; } 100% { margin-top: 60px; height: 35px; width: 75px; } } .ball { animation: fall 1s ease-in infinite alternate; }
